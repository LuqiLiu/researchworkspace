# Backup and restore

The Stage 0 backup covers PostgreSQL and the persistent media volume. Static files are generated by `collectstatic` and do not need to be backed up.

## Create a backup

Run from the repository root on the production host:

```bash
./scripts/backup.sh
```

The script creates a timestamped directory under `BACKUP_ROOT` (default: `./backups`) containing:

- `database.sql.gz`
- `media.tar.gz`
- `manifest.txt`

Keep backup directories outside any path served by Caddy. Copy them to separate encrypted storage and apply the retention policy described in the PRD.

## Restore

Restoring replaces the current database schema and media contents. Stop user traffic and verify the selected backup directory first:

```bash
./scripts/restore.sh ./backups/20260101T120000Z --yes
```

After restoration, the script runs migrations. Start the stack and verify:

```bash
docker compose -f compose.yml up -d
curl --fail https://your-domain.example/health/ready/
```

## Required recovery exercise

Before production launch, restore a backup into an empty test environment, sign in to the administrator site, and record the date and outcome. A backup is not considered valid until a restore has succeeded.
