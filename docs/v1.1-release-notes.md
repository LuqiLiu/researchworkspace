# Research Workspace Lite V1.1

V1.1 adds a secure image workflow for experiment records and improves Markdown
math rendering. It also makes outgoing collaboration access fully visible to
content owners.

## Sharing visibility

- Add a dedicated **我发出的分享** overview from the collaboration navigation.
- List the concrete users who can access each article, idea, experiment, or
  other research record.
- Show whether access comes from a direct grant, project membership, or both.
- Merge duplicate access routes into one row and show the highest effective
  permission.
- Show whether each recipient can open attachments or only the Markdown body.
- Keep the complete access roster owner-only; collaborators cannot inspect who
  else received the record.

## Experiment images

- Upload PNG, JPEG, GIF, and WebP images from an experiment detail page.
- Paste, drag and drop, or select multiple images while editing a saved record.
- Insert a protected Markdown image reference at the current cursor position.
- Preview images in a responsive gallery, open the original, download it, or
  delete it as the owner.
- Validate raster signatures instead of trusting file extensions or browser
  MIME metadata. SVG and disguised HTML are not served inline.
- Apply the existing attachment permission matrix to every image request and
  disable shared caching so revocation takes effect immediately.
- Preserve image files during workspace export and rewrite Markdown references
  to portable relative archive paths.

## Markdown math

- Render `$...$` and `\\(...\\)` inline expressions.
- Render `$$...$$` and `\\[...\\]` display expressions.
- Keep code blocks, inline code, and editor textareas untouched.

No database migration is required for V1.1. Existing attachments remain
compatible and safe raster images automatically appear in the image gallery.
